## Fix Strategy

### Fix 1: KeyEventHandler.java
Add `Hangul_initial` and `Hangul_medial` cases to `key_up()`.

For `Hangul_initial`:
- If no active composition: start new Hangul composition, send as composing text
- If active composition with initial+medial: treat as final consonant

For `Hangul_medial`:
- If active composition with initial: combine into precomposed syllable
- If no active composition: treat as standalone vowel

Requires tracking composition state (initial_idx, medial_idx, final_idx).

### Fix 2: KeyModifier.java
In `combine_hangul_initial()`, handle `case Hangul_initial:` properly:
- If the new key is the SAME as current initial: output standalone initial jamo
- If different: may need to handle as final or as new syllable start

Current code greys out ALL Hangul_initial keys when latched, which is wrong.
